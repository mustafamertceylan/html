📏 Relative Font Sizes

% (yüzde) · em · rem — Neden absolute (px) yerine relative birimler kullanmalıyız?

🤔 Neden Relative Birimler Kullanmalıyız?

Absolute birimler (px) yerine relative birimler (%, em, rem) kullanmanın avantajları:

  • ✅ Farklı ekran boyutlarında çalışır — Responsive tasarım için idealdir
  • ✅ Daha kolay hesaplanır — Magic numbers yerine oranlar kullanılır
  • ✅ Erişilebilirlik (Accessibility) — Kullanıcı tarayıcı zoom'u yaptığında orantılı büyür
  • ✅ Daha az kod — Bir yerde değiştir, her yere yansısın
/* Absolute (px) — Magic numbers */
.container {
  width: 122px;
  margin: 122px;
  font-size: 122px;
}

/* Relative (%) — Oranlar */
.container {
  width: 10%;
  margin: 10%;
  font-size: 100%;
}
📌 Örnek Karşılaştırma:

10% / 80% / 10% vs 122px / 926px / 122px

Relative birimlerle çalışmak, absolute birimlerle çalışmaktan çok daha kolaydır!

📌 px (Piksel) — Mutlak Birim (Absolute)

px en temel birimdir. Ekran pikseli cinsinden mutlak bir değerdir. Ancak responsive tasarımda esnek değildir.

/* px kullanımı */
h1 {
  font-size: 32px;  /* Her ekranda 32px */
}

Bu yazı 32px (mutlak)

Bu yazı 16px (mutlak)

⚠️ px dezavantajı: Kullanıcı tarayıcı zoom'u yapsa bile yazı boyutu değişmez. Bu, erişilebilirlik açısından sorun oluşturabilir!

📊 % (Yüzde) — Parent Element'e Göre

Yüzde (%) birimi, font-size değerini ebeveyn (parent) elementin font-size'ine göre hesaplar. font-size: 100% = ebeveyn ile aynı boyut.

.parent {
  font-size: 30px;
}

.child {
  font-size: 75%;  /* 30px * 0.75 = 22.5px */
}
Parent: font-size: 30px
Child: font-size: 75% → 30px × 0.75 = 22.5px

📌 İç İçe Yüzde — Bileşik Etki (Compounding)

Parent: font-size: 200% (varsayılan 16px × 2 = 32px)

Child: font-size: 75% → 32px × 0.75 = 24px

⬆️ Bileşik etki: 16px × 200% × 75% = 24px

📐 em — Element'in Kendi Font-Size'i

em, element'in kendi hesaplanmış font-size'ini temsil eder. 1em = inherit edilen font-size (yani %100 ile aynı).

/* em kullanımı */
.parent {
  font-size: 30px;
}

.child {
  font-size: 0.75em;  /* 30px * 0.75 = 22.5px */
}
Parent: font-size: 30px
Child: font-size: 0.75em → 30px × 0.75 = 22.5px

📌 İç İçe em — Bileşik Etki (Compounding)

⚠️ ÖNEMLİ: em, iç içe kullanıldığında bileşik etki (compounding) yaratır!
Parent: font-size: 2em (varsayılan 16px × 2 = 32px)

Child: font-size: 0.75em → 32px × 0.75 = 24px

Deep child: 0.5em → 24px × 0.5 = 12px

⬆️ Bileşik etki: 16px × 2 × 0.75 × 0.5 = 12px

📌 h1 varsayılanı 2em:

h1 varsayılan font-size: 2em (32px)

Tarayıcılar h1'e varsayılan olarak 2em verir. Bu nedenle 16px root'ta h1 32px görünür.

📐 rem — Root EM (HTML Elementi)

rem (root em), root element'in (html tag) font-size'ini temsil eder. 1rem = html elementinin font-size'i. Bileşik etki YOKTUR!

/* rem kullanımı — html root belirlenir */
html {
  font-size: 16px;  /* Varsayılan */
}

.child {
  font-size: 0.75rem;  /* 16px × 0.75 = 12px — parent'dan bağımsız! */
}
Root font-size: 16px (varsayılan)
Child: font-size: 0.75rem → 16px × 0.75 = 12px (parent'dan bağımsız!)

📌 İç İçe rem — Bileşik Etki YOK!

✅ rem'in en büyük avantajı: İç içe kullanılsa bile bileşik etki yaratmaz!
Parent: font-size: 2rem (16px × 2 = 32px)

Child: font-size: 0.75rem → HALA 12px! (parent'dan etkilenmez)

Deep child: 0.5rem → HALA 8px! (parent'dan etkilenmez)

⬆️ rem'de bileşik etki yok! Her zaman root'a göre hesaplanır.

📊 px vs % vs em vs rem — Karşılaştırma

BirimReferansBileşik EtkiKullanım Alanı
px Ekran pikseli (mutlak) Yok Border, shadow, sabit boyutlar
% Ebeveyn (parent) element ✅ Var Responsive genişlik, yükseklik
em Element'in kendi font-size'i ✅ Var Font-size, margin, padding (iç içe istendiğinde)
rem Root (html) font-size ❌ Yok Font-size için EN ÇOK ÖNERİLEN

🎯 em/rem Nerelerde Kullanılır?

Sadece font-size için değil, birçok CSS özelliğinde em/rem kullanabilirsiniz:

/* em/rem kullanılabilecek özellikler */
.element {
  font-size: 1.2rem;      /* ✅ Font-size için rem önerilir */
  line-height: 1.5em;     /* ✅ Satır yüksekliği için em */
  margin-top: 1rem;       /* ✅ Margin için rem */
  margin-bottom: 1.5rem;  /* ✅ Margin için rem */
  padding: 1rem;          /* ✅ Padding için rem */
  border-radius: 0.5rem;  /* ✅ Border-radius için rem */
  width: 80%;             /* ✅ Genişlik için yüzde */
  max-width: 1200px;      /* ⚠️ Max-width için px veya rem */
}
🎯 En İyi Pratikler:
  • ✅ font-size için → rem (bileşik etki yok, tutarlı)
  • ✅ margin / padding için → rem (tutarlı boşluklar)
  • ✅ line-height için → em veya birimsiz değer (1.5)
  • ✅ width / height için → % veya vw/vh
  • ✅ border / shadow için → px (ince detaylar)

💡 Pratik Örnek: Responsive Tipografi

/* Root font-size değişince her şey orantılı değişir! */
html {
  font-size: 16px;  /* Varsayılan */
}

/* Tablet için */
@media (min-width: 768px) {
  html {
    font-size: 18px;  /* Biraz daha büyük */
  }
}

/* Masaüstü için */
@media (min-width: 1200px) {
  html {
    font-size: 20px;  /* Daha da büyük */
  }
}

/* Tüm font'lar rem ile tanımlandığı için otomatik ölçeklenir! */
h1 { font-size: 2rem; }   /* 32px → 36px → 40px */
h2 { font-size: 1.5rem; } /* 24px → 27px → 30px */
p { font-size: 1rem; }    /* 16px → 18px → 20px */

⚠️ Sık Yapılan Hatalar

❌ Hata 1: em'i iç içe kullanıp istenmeyen bileşik etki yaratmak 
.parent { font-size: 2em; }    /* 32px */
.child { font-size: 0.5em; }   /* 16px BEKLERKEN → 16px DEĞİL! */
/* Aslında: 32px × 0.5 = 16px OLUR! (doğru aslında) */
/* Ama derinleştikçe çarpım etkisi artar! */
✅ Çözüm: İç içe elementlerde font-size için rem kullanın!
❌ Hata 2: html root font-size'i değiştirip rem kullanan elementleri unutmak 
html { font-size: 10px; }  /* 10px yapınca tüm rem'ler küçülür! */
/* 1rem = 10px oldu! */
✅ Çözüm: Root font-size değiştirirken tüm rem birimlerinin buna göre hesaplanacağını unutmayın!
❌ Hata 3: Erişilebilirliği unutmak 
/* Kullanıcı tarayıcı zoom'u yapınca px büyümez! */
body { font-size: 16px; }  /* Zoom'da değişmez */
✅ Çözüm: rem kullanın — kullanıcı zoom yapınca orantılı büyür!

📋 Relative Font Sizes — Özet

  • 📌 px → Mutlak birim, responsive değil, erişilebilirlik sorunlu
  • 📊 % (yüzde) → Ebeveyne göre, bileşik etki var
  • 📐 em → Element'in kendi font-size'ine göre, bileşik etki var
  • 📏 rem → Root (html) font-size'ine göre, bileşik etki YOK
  • 🎯 font-size için en çok önerilen → rem! (tutarlı, öngörülebilir)
  • 📏 margin/padding için → rem (tutarlı boşluklar)
  • 📐 line-height için → em veya birimsiz
  • ♿ Erişilebilirlik için relative birimler şart! (kullanıcı zoom yapabilir)
