📱 Mobil Web & Responsive Tasarım

Viewport Meta Tag · CSS Media Queries · Chrome Device Mode · Mobile-First Design

📱 Mobil Web Problemi

Mobil cihazlarda (telefon, tablet) web sitelerini görüntülerken karşılaştığımız temel sorun: ekran çok küçük!

📌 Mobil Tarayıcılar Nasıl Çalışır?

Mobil tarayıcılar, web sayfalarını önce masaüstü genişliğinde (~1000px) render eder, sonra tüm sayfa ekrana sığana kadar "zoom out" (uzaklaştırma) yapar. Bu nedenle yazılar çok küçük görünür!

⚠️ Sorun: Mobil desteği eklenmemiş sitelerde yazılar çok küçük ve okunması zor olur.
✅ Çözüm: Responsive Web Design — Farklı ekran boyutlarında iyi görünen CSS yazma sanatı.

🔧 Viewport Meta Tag — Mobil Görüntülemenin Anahtarı

Viewport meta tag, mobil tarayıcılara sayfanın nasıl ölçekleneceğini söyler. Bu tag olmadan mobil cihazlar sayfayı masaüstü genişliğinde render eder!

<!-- HTML head bölümüne mutlaka ekleyin! -->
<meta name="viewport" content="width=device-width, initial-scale=1">
📌 Parametrelerin Anlamı:
  • name="viewport" → "Tarayıcı, sana viewport'u nasıl göstermemi istediğimi söyleyeceğim."
  • width=device-width → "Viewport'un genişliği, cihazın genişliği ile aynı olsun." (Masaüstü genişliğinde render etme!)
  • initial-scale=1 → "Başlangıç zoom seviyesi %100 olsun."
🎯 Altın Kural: Bu tag'i her HTML sayfanıza ekleyin! Modern web geliştirmenin vazgeçilmezidir.

📏 CSS Media Queries — Farklı Ekranlar İçin Farklı Stiller

Media Queries, cihazın özelliklerine (ekran genişliği, yüksekliği, yönü vb.) göre farklı CSS kuralları uygulamanızı sağlar.

/* Ekran genişliği 500px veya daha küçükse */
@media (max-width: 500px) {
  article {
    margin: 0 2px;
  }
}

/* Ekran genişliği 768px ile 1024px arasındaysa */
@media (min-width: 768px) and (max-width: 1024px) {
  .sidebar {
    display: none;
  }
}

/* Yatay mod (landscape) */
@media (orientation: landscape) {
  .hero {
    height: 100vh;
  }
}

🛠️ Chrome Device Mode — Mobil Simülasyon

Chrome Device Mode, tarayıcınızda farklı mobil cihazları simüle etmenizi sağlayan bir geliştirici aracıdır.

🎯 Nasıl Açılır?
  1. F12 veya Sağ Tık → İncele ile Geliştirici Araçları'nı açın
  2. 📱 Cihaz simgesine (Toggle Device Toolbar) tıklayın
  3. Veya Ctrl + Shift + M (Windows/Linux) / Cmd + Shift + M (Mac)
📌 Chrome Device Mode Avantajları:
  • ✅ Çok kullanışlı — Tek tıkla mobil görünüm
  • ✅ Genellikle doğru — Gerçek cihaza yakın sonuçlar
  • ✅ Farklı cihazlar — iPhone, Pixel, iPad vb. seçenekler
  • ✅ Responsive mod — Ekranı istediğiniz boyuta sürükleyin
⚠️ Chrome Device Mode Dezavantajları:
  • ❌ Her zaman %100 doğru değil — Özellikle iPhone'da farklılıklar olabilir
  • ❌ Performans sorunlarını simüle etmez — Gerçek cihazda daha yavaş olabilir
  • ❌ Bazı özellikler hatalı çalışabilir
💡 En İyi Pratik: Geliştirme sırasında Chrome Device Mode kullanın, ancak her zaman gerçek cihazlarda da test edin!

🔌 Remote Debugging — Gerçek Cihazda Test

📱 Android (Chrome Remote Debugging)

1. Android telefonunuzda USB Hata Ayıklama'yı açın
2. USB kablo ile bilgisayara bağlayın
3. Chrome'da chrome://inspect adresini açın
4. Telefonunuzdaki sayfayı inceleyin ve debug edin!

📱 iOS (Safari Remote Debugging)

1. iPhone'da Ayarlar → Safari → Geliştirici → Web Denetçisi'ni açın
2. USB kablo ile Mac'e bağlayın
3. Mac'te Safari → Geliştirici menüsünden telefonunuzu seçin
💡 Port Forwarding ile Local Server:

Bilgisayarınızda çalışan local sunucuyu (localhost:3000) telefonunuzda test etmek için Chrome remote debugging üzerinden port forwarding kullanabilirsiniz.

📱 Mobile-First vs Desktop-First Tasarım

📌 Mobile-First (Önerilen):

Önce mobil için CSS yaz, sonra min-width ile daha büyük ekranlar için stiller ekle.

/* Mobil varsayılan (max-width yok) */
.container {
  display: flex;
  flex-direction: column;
}

/* Tablet ve üzeri */
@media (min-width: 768px) {
  .container {
    flex-direction: row;
  }
}
📌 Desktop-First (Eski yaklaşım):

Önce masaüstü için CSS yaz, sonra max-width ile küçük ekranlar için stiller ekle.

/* Masaüstü varsayılan */
.container {
  display: flex;
  flex-direction: row;
}

/* Mobil */
@media (max-width: 768px) {
  .container {
    flex-direction: column;
  }
}
🎯 Neden Mobile-First?
  • 📱 Mobil kullanım masaüstünden fazla
  • ⚡ Daha hızlı yüklenir (mobil için daha az CSS)
  • 🎨 Önce içerik, sonra düzen düşünmeyi sağlar

💡 Pratik İpuçları

  • 📱 Her zaman viewport meta tag'i ekleyin! — Unutmayın, olmazsa mobilde zoom yapılır.
  • 📏 Rem/em kullanın — Piksel yerine göreceli birimler responsive tasarımı kolaylaştırır.
  • 🖼️ max-width: 100% — Resimlerin taşmasını engeller.
  • 🎯 Mobile-first yaklaşımı benimseyin — Önce küçük ekran, sonra büyük ekran.
  • 🔍 Gerçek cihazlarda test edin — Simülasyon her zaman %100 doğru değildir.
  • 📐 Flexbox ve Grid kullanın — Responsive düzenler için en iyi araçlar.
