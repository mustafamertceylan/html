🏷️ Class ve ID Seçiciler

CSS seçicilerin gücü — Element, Class ve ID ile stillendirme

📖 3 Temel CSS Seçici Tipi

SeçiciSözdizimiAçıklama
Element SeçicipTüm <p> elementlerini seçer
ID Seçici#abcid="abc" olan elementi seçer (benzersiz)
Class Seçici.abcclass="abc" olan elementleri seçer
 
<h1 id="title">Homework</h1>
<em class="hw">HW0</em> is due Friday.<br/>
<em class="hw">HW1</em> goes out Monday.<br/>
<em>All homework due at 11:59pm.</em>
      
/* Class seçici - tüm .hw class'ına sahip elementleri seçer */
.hw {
  color: hotpink;
}

/* ID seçici - #title id'sine sahip elementi seçer */
#title {
  color: purple;
}

🔍 Class vs ID — Temel Farklar

ÖzellikClass (.)ID (#)
Kullanım sıklığıBirden fazla elementte kullanılabilirSayfada sadece 1 elementte kullanılabilir
Benzersizlik❌ Benzersiz olmak zorunda değil✅ Benzersiz olmalı
Öncelik (Specificity)Orta (10 puan)Yüksek (100 puan)
Birden fazla class✅ Evet (boşlukla ayrılır)❌ Hayır (tek ID)
 
/* class: birden fazla elementte kullanılabilir */
.highlight {
  background-color: yellow;
}

/* id: sadece bir elementte kullanılmalı! */
#main-title {
  font-size: 2rem;
}

/* Birden fazla class - boşlukla ayrılır */
<span class="hw new">HW1</span>
💡 Altın Kural:
  • Class (.) → Birden fazla elemente aynı stili vermek için
  • ID (#) → Sayfadaki benzersiz bir elementi seçmek için

🔷 element.className Sözdizimi

Belirli bir element tipindeki belirli bir class'a sahip elementleri seçmek için kullanılır.

/* Sadece <p> elementleri içindeki .highlight class'ına sahip olanlar */
p.highlight {
  background-color: yellow;
  font-weight: bold;
}

📺 Örnek

Bu paragraf p.highlight ile seçildi

Bu div aynı class'a sahip ama seçilmedi (p değil)
✅ Özet: p.abc → "abc class'ına sahip <p> elementleri"

🔽 Descendent Seçici (Alt Eleman)

Descendent seçici, bir elementin içindeki (herhangi bir seviyede) başka bir elementi seçer.

/* div içindeki tüm strong elementleri seçer */
div strong {
  color: red;
}

📺 Örnek

Bu bir strong elementi (div içinde → seçilir)

Bu strong div dışında → seçilmez
📌 Önemli: Elementin doğrudan çocuk (direct child) olması gerekmez! Çok derinlerde de olsa seçilir.

🎯 Pratik Kullanım: Grup Kapsayıcı

<div class="announcement">
  <h2>Duyuru</h2>
  <p>Yarın ders yok!</p>
  <strong>Önemli:</strong> Sınav ertelendi.
</div>

.announcement {
  border: 1px solid blue;
  padding: 1rem;
}

.announcement strong {
  color: red;  /* Sadece announcement içindeki strong'lar */
}

📋 Virgül ile Gruplama (Selector, Selector)

Birden fazla seçiciye aynı stil kurallarını uygulamak için virgül kullanılır.

/* <h2> elementleri VE <div> elementleri */
h2, div {
  font-family: Arial;
  margin-bottom: 10px;
}

⚠️ Sık Yapılan Hatalar

❌ Hata 1: p.abc vs p .abc 
p.abc   → "abc class'lı <p> elementi"
p .abc  → "<p> içindeki abc class'lı element"
❌ Hata 2: p .abc vs p, .abc 
p .abc  → "<p> içindeki abc class'lı element"
p, .abc → "Tüm <p> elementleri VE tüm abc class'lı elementler"

📺 Görsel Fark

p.abc → bu element seçilir

p .abc → bu span seçilir (p içinde)

🔗 Birleştirilmiş Seçiciler

#main li.important strong {
  color: red;
}

Bu seçiciyi sağdan sola oku:

  1. <strong> etiketlerini seç
  2. ... "important" class'ına sahip <li> etiketlerinin içindeki
  3. ... "main" id'sine sahip elementin içindeki

📺 Örnek

  • Bu li seçilir (önemli strong)
  • Bu li seçilmez (class yok)
Bu strong seçilmez (li içinde değil)

⚖️ Specificity — Stiller Çakıştığında

Aynı element için birden fazla CSS kuralı varsa, en spesifik (özel) olan kural kazanır.

div strong { color: red; }   /* Daha spesifik */
strong { color: blue; }      /* Daha az spesifik */
Bu strong elementi kırmızı mı mavi mi?

⬆️ div strong daha spesifik olduğu için KIRMIZI olur!

📊 Specificity Sıralaması (Öncelik)

  1. ID (#) — En yüksek öncelik (100 puan)
  2. Class (.) — Orta öncelik (10 puan)
  3. Element (div, p, etc.) — Düşük öncelik (1 puan)
#title { color: red; }      /* 100 puan - KAZANIR */
.title { color: blue; }     /* 10 puan */
h1 { color: green; }        /* 1 puan */
📌 Aynı specificity'de son yazılan kazanır: 
strong { color: red; }
strong { color: blue; }  /* Son yazılan → MAVİ kazanır */
📖 Cascade Nedir? "Cascading Style Sheets" kelimesindeki C, stillerin çakıştığında hangi kuralın uygulanacağını belirleme sürecini tanımlar.

🧬 Inheritance — Stillerin Kalıtımı

Birçok CSS özelliği ebeveynden (parent) çocuğa (child) kalıtım yoluyla geçer.

body {
  font-family: Helvetica, sans-serif;
}

/* body içindeki tüm elementler Helvetica font'u alır */

📺 Kalıtım Örneği

Bu paragraf Helvetica font'u kalıtım yoluyla aldı.

Div de Helvetica aldı.
Span da Helvetica aldı.

⚠️ Tüm özellikler kalıtım yoluyla geçmez!

a {
  display: block;
  font-family: Arial;
}

<a href="#">
  Back to <em>Home</em>
</a>
  • ✅ font-family → kalıtım yoluyla geçer (Home da Arial olur)
  • ❌ display → kalıtım yoluyla geçmez (em inline kalır)
📌 Genel kural:
  • ✅ Metin ile ilgili özellikler (font-family, color, text-align) → kalıtım geçer
  • ❌ Düzen (layout) ile ilgili özellikler (display, margin, padding, border) → kalıtım geçmez
💡 İpucu: Bir özelliğin kalıtım alıp almadığını MDN (Mozilla Developer Network)'den öğrenebilirsiniz!

📋 Genel Özet

  • 🏷️ Class (.) → Birden fazla elementte kullanılır, stilleri gruplamak için
  • 🆔 ID (#) → Benzersizdir, sayfada sadece bir kez kullanılır
  • 📦 Element seçici → Tüm elementleri seçer (p, div, h1 vb.)
  • 🔽 Descendent seçici → Bir elementin içindeki elementleri seçer (div strong)
  • 📋 Virgül ile gruplama → Birden fazla seçiciye aynı stili uygular (h2, div)
  • ⚖️ Specificity → ID > Class > Element (en spesifik kazanır)
  • 🧬 Inheritance → Metin özellikleri kalıtım yoluyla geçer, düzen özellikleri geçmez
  • 📚 MDN → En iyi referans kaynağı!
