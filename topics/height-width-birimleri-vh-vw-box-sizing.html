📏 Height & Width Birimleri

vh, vw, % (yüzde) ve box-sizing — Boyutlandırmanın temelleri

📊 Yüzde (%) Birimleri — Ebeveyne Göre Boyutlandırma

Yüzde (%) birimleri, bir elementin boyutunu ebeveyn (parent) elementin boyutuna göre belirler.

/* width: yüzde → ebeveynin GENİŞLİĞİNE göre */
width: 50%;   /* Ebeveyn genişliğinin %50'si */

/* height: yüzde → ebeveynin YÜKSEKLİĞİNE göre */
height: 50%;  /* Ebeveyn yüksekliğinin %50'si */

📺 Yüzde Örneği

width: 50%
height: 50%

⬆️ Parent: 400x200 → Child: 200x100 (50%)

⚠️ ÖNEMLİ — Height yüzdesi için!

Bir elemente height: 50% verebilmek için, ebeveynin yüksekliği açıkça belirtilmiş olmalıdır! Aksi halde yüzde çalışmaz.

/* Bu ÇALIŞIR */
.parent {
  height: 400px;     /* Açıkça belirtilmiş */
}
.child {
  height: 50%;       /* 200px olur */
}

/* Bu ÇALIŞMAZ */
.parent {
  /* height belirtilmemiş → auto */
}
.child {
  height: 50%;       /* ÇALIŞMAZ! */
}

🖥️ Viewport — Tarayıcı Görüntü Alanı

Viewport, web sayfasının göründüğü dikdörtgen alandır. Kaydırma çubuğu (scrollbar) ile bu alan içinde hareket edebilirsiniz.

📌 Tarayıcı Kelime Dağarcığı:
  • Viewport → Web sayfasının gösterildiği dikdörtgen alan (scrollable)
  • Chrome (tarayıcı arayüzü) → Viewport dışındaki tüm UI (sekme çubuğu, adres çubuğu, bookmark bar, vb.)
🎯 Görselleştirme:

├─────────────────────────────────────────────────────────────┤

📐 vh ve vw — Viewport'a Göre Boyutlandırma

vh (viewport height) ve vw (viewport width), elementleri viewport'un boyutuna göre boyutlandırmanızı sağlar.

/* 1vh = viewport yüksekliğinin 1/100'ü */
/* 1vw = viewport genişliğinin 1/100'ü */

height: 100vh;   /* Viewport yüksekliğinin %100'ü (tam ekran) */
width: 100vw;    /* Viewport genişliğinin %100'ü */
height: 50vh;    /* Viewport yüksekliğinin yarısı */
width: 50vw;     /* Viewport genişliğinin yarısı */

📺 vh Örneği (30vh)

Bu kutu height: 30vh

⬆️ Yüksekliği, viewport yüksekliğinin %30'u kadar!

📺 vw Örneği (30vw)

Bu kutunun genişliği 30vw

⬆️ Genişliği, viewport genişliğinin %30'u kadar!

✨ vh/vw vs % Farkı:
% (yüzde) → Ebeveyn elemente göre hesaplanır
vh/vw → Viewport'a göre hesaplanır (ebeveynden bağımsız!)

🎯 vh ile Tam Sayfa Düzeni — Header + Content + Footer

vh kullanarak, header ve footer'ı viewport'un üst ve altına sabitleyen bir düzen oluşturabiliriz.

/* Tam sayfa düzeni için flexbox + vh */
body {
  display: flex;
  flex-direction: column;
  height: 100vh;      /* Body, viewport'un tamamı */
  margin: 0;
}

.header { flex-shrink: 0; }  /* Sabit yükseklik */
.main { flex-grow: 1; }      /* Kalan tüm alanı kaplar */
.footer { flex-shrink: 0; }  /* Sabit yükseklik */
📌 HEADER (Sabit — Her zaman üstte)
📄 MAIN CONTENT
(Kalan tüm alanı kaplar — flex-grow: 1)
© FOOTER (Sabit — Her zaman altta)
💡 Bu teknikle:
  • Header her zaman viewport'un en üstünde kalır
  • Footer her zaman viewport'un en altında kalır
  • Ana içerik, kalan tüm alanı esnek şekilde kaplar

📦 box-sizing — Kutu Modeli Hesaplama Şekli

box-sizing, bir elementin width ve height özelliklerinin nasıl hesaplanacağını belirler.

/* content-box (Varsayılan) */
box-sizing: content-box;
/* width ve height SADECE içerik alanını belirler */
/* padding ve border EKLENİR → Kutu büyür! */

/* border-box (ÖNERİLEN) */
box-sizing: border-box;
/* width ve height = content + padding + border */
/* padding ve border EKSTRA ALAN EKLEMEZ! */

📺 content-box vs border-box

content-box (Varsayılan)
width: 200px
padding: 20px
border: 5px

Toplam genişlik: 250px
(200 + 40 + 10)

border-box (Önerilen)
width: 200px
padding: 20px
border: 5px

Toplam genişlik: 200px
(padding ve border içeriye alınır)

🎯 Neden border-box hayat kurtarır?
  • Bir elemente width: 200px verdiğinizde, padding veya border ekleseniz bile kutu 200px kalır
  • Responsive tasarımda hesaplama derdi yok!
  • En iyi pratik: Tüm elementlere box-sizing: border-box uygulayın!
/* Tüm elementler için border-box — Modern CSS standardı! */
*, *::before, *::after {
  box-sizing: border-box;
}

⚠️ Box Model + Border — Klasik Sorun

Varsayılan content-box kullanırken, bir elemente border eklemek kutunun toplam boyutunu değiştirir!

/* content-box ile — border ekleyince kutu büyür! */
.box {
  width: 200px;
  border: 10px solid red;
  /* Toplam genişlik: 220px! (200 + 10 + 10) */
}
width: 200px
border: 10px

⬆️ İçerik genişliği 200px, ama toplam genişlik 220px!

✅ border-box ile çözüm: 
.box {
  width: 200px;
  border: 10px solid red;
  box-sizing: border-box;
  /* Toplam genişlik: 200px! (border içeriye alınır) */
}
width: 200px
border: 10px
box-sizing: border-box

⬆️ Toplam genişlik 200px kalır!

🎨 Pratik Örnek: Tam Ekran Hero Bölümü

.hero {
  height: 100vh;           /* Viewport'un tamamı */
  width: 100%;             /* Tam genişlik */
  display: flex;
  flex-direction: column;
  justify-content: center;
  align-items: center;
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
  color: white;
  text-align: center;
}

.hero h1 {
  font-size: 4rem;
  margin-bottom: 1rem;
}

.hero p {
  font-size: 1.2rem;
  max-width: 600px;
}

Bu bölüm 50vh yüksekliğinde! (Tam ekran için 100vh kullanın)

📋 Özet

  • 📊 % (yüzde) → Ebeveyn elemente göre boyutlandırma (height için ebeveynin height'i belirtilmeli!)
  • 🖥️ Viewport → Sayfanın göründüğü dikdörtgen alan
  • 📐 vh/vw → Viewport'a göre boyutlandırma (ebeveynden bağımsız!)
  • 🎯 100vh → Tam ekran yüksekliği (header+footer+content = tam sayfa)
  • 📦 box-sizing: border-box → Padding ve border'ı width/height içine alır (önerilen)
  • ⚠️ content-box (varsayılan) → Padding ve border eklenir, kutu büyür!
  • ✨ En iyi pratik: *, *::before, *::after { box-sizing: border-box; }
